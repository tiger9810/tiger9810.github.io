## なぜこの方法が必要なのか？

AIとコーディングをしていると、

- 「1つのチャットに複数の問題を詰め込みすぎて、AIが混乱している」
- 「会話が長くなりすぎて、何を聞いていたか分からなくなった」
- 「複数のトラブルを同時に解決しようとして、逆に時間がかかった」

**この方法を使えば、そんな悩みが改善するかもしれません。**

---

## この方法の3つのメリット

### メリット1: AIの回答精度が向上
1つのチャットに1つの問題だけを投げることで、AIは余計な情報に惑わされず、的確な回答を返せます。

### メリット2: トラブルの優先順位が明確になる
すべてのトラブルをSOWにまとめることで、「何から解決すべきか」が一目瞭然になります。

### メリット3: 複数のトラブルを並行処理できる
複数のチャットを同時に開けば、待ち時間ゼロで効率的に問題解決できます。

---

## 手法の全体像（2ステップ）
```
ステップ1: 
・メインチャットでトラブルを報告
・AIにSOWを作成してもらう
・別チャットに投げるプロンプトを作成してもらう
    ↓
ステップ2: 別チャットで解決→メインチャットに報告
```

---

## ステップ1: メインチャットでトラブルを報告

### やること
発生しているトラブルを**箇条書きで**メインチャットに報告します。

### プロンプト例
```
現在、予約管理システムを開発中ですが、以下のトラブルが発生しています。

トラブル1: 新規顧客登録ボタンを押しても「登録に失敗しました」と出る
トラブル2: 空いている時間なのに「全スタッフが埋まっています」と表示される
トラブル3: 60分予約なのに画面では30分分しか表示されない

これらのトラブルをSOWにまとめて、優先順位をつけてください。
その後、各トラブルを別チャットで解決したいので、別チャットに投げるための明確で完結したトラブル別のプロンプトを作成してください。
まずはトラブル1のプロンプトをお願いします。
```

### ポイント
- **具体的に**書く（スクリーンショットも添付するとベター）
- **複数のトラブルをまとめて**報告してOK
- 「どうすればいいか分からない」という状態でも大丈夫

---


### AIが作成するプロンプトの特徴
- **文脈を含む**: 「予約システム開発中」など背景情報
- **問題が明確**: 何が起きているか、何をしたいか
- **コードを含む**: 現在のコードを提示
- **具体的な質問**: 「どう修正すればいいか」を明示

### AIが作成したプロンプト例
```
React + TypeScriptで予約管理システムを開発しています。
予約画面から新規顧客を登録する機能で「顧客登録に失敗しました」というエラーが出ます。

## 現在の状況
BookingForm.tsxからCustomerFormをモーダル内で呼び出して新規顧客登録を試みていますが失敗します。

## 関連コード
// BookingForm.tsx（一部抜粋）
const handleCreateCustomer = async (data: CustomerCreate) => {
  try {
    const newCustomer = await customerService.create(data);
    setCustomers([...customers, newCustomer]);
    setValue('customer_id', newCustomer.id);
    setIsCustomerModalOpen(false);
    alert('顧客を登録しました');
  } catch (error) {
    alert('顧客登録に失敗しました');
  }
};

## 型定義
export interface CustomerCreate {
  display_name: string;
  phone: string;
  email?: string;
  notes?: string;
}

## 問題
CustomerFormにcustomerプロパティを渡していないためエラーが発生している可能性があります。

## やりたいこと
1. CustomerFormを新規登録と編集の両方で使えるようにする
2. customerプロパティをオプショナルにする
3. 新規登録時はフォームを空にし、編集時は既存データを表示する

## 質問
CustomerForm.tsxをどのように修正すれば、新規登録と編集の両方に対応できますか？
完全なコードを提示してください。
```

## ステップ2: 別チャットで解決→メインチャットに報告

### やること

1. **新しいチャットを開く**
2. **ステップ1で作成したプロンプトをコピペ**
3. **AIの回答を受け取る**
4. **コードを実装**
5. **メインチャットに「トラブル1解決しました」と報告**

### メインチャットへの報告例
```
トラブル1: 新規顧客登録の失敗は解決しました。
次はトラブル2に移行します。
解決を別チャットで依頼するための、明確で完結したプロンプトを作成してください。
```

### この流れを繰り返す
```
トラブル1解決 → トラブル2のプロンプト作成 → 別チャットで解決
→ トラブル2解決 → トラブル3のプロンプト作成 → 別チャットで解決
→ 完了！
```

## 実践例：予約システム開発での使用例

### メインチャットでの会話
```
【あなた】
以下のトラブルが発生しています。SOWを作成してください。

トラブル1: 新規顧客登録が失敗する
トラブル2: 空き時間判定が誤っている
トラブル3: 予約マスが30分分しか表示されない

トラブル1から解決したいので、別チャット用のプロンプトを作成してください。

【AI】
（詳細なSOWを作成）
優先順位:
1. トラブル2（空き時間判定）- 優先度: 高
2. トラブル3（予約マス表示）- 優先度: 中
3. トラブル1（新規顧客登録）- 優先度: 低

（完結したプロンプトを作成）

【あなた】
（別チャットで解決）
トラブル1解決しました。次はトラブル2のプロンプトを作成してください。

【AI】
（トラブル2用のプロンプトを作成）
...トラブルが解消するまで行う
```

## この方法が効果的な理由（技術的背景）

### 1. コンテキストの分離
AIは会話が長くなると、過去の情報に引きずられて精度が下がります。
1チャット1問題にすることで、常に最高精度の回答を得られます。

### 2. 問題の構造化
SOWを作成することで、「何が問題か」「何から解決すべきか」が明確になります。
これはAIだけでなく、あなた自身の思考整理にも役立ちます。

### 3. 並行処理の実現
複数のチャットを開けば、待ち時間なく次々と問題を解決できます。

## よくある質問（FAQ）

### Q1: SOWって難しそう...自分で書くんですか？
**A**: いいえ、AIが作成します。あなたは「トラブルを報告する」だけでOKです。

### Q2: メインチャットと別チャット、どう使い分けるの？
**A**: 
- **メインチャット**: プロジェクト全体の管理、SOW作成、優先順位決定
- **別チャット**: 個別トラブルの解決（1チャット1問題）

### Q3: 複数のチャットを開くのは面倒では？
**A**: 最初は手間に感じるかもしれませんが、1つのチャットで混乱するより圧倒的に速いです。

### Q4: すべてのトラブルでこの方法を使うべき？
**A**: いいえ。簡単なトラブル（タイプミスなど）は普通に聞いてOKです。
複数のトラブルが絡み合っている場合や、優先順位をつけたい場合に使ってください。
